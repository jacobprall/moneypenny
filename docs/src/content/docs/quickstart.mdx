## Prerequisites

- **Rust toolchain** — install via [rustup](https://rustup.rs) if you don't have it
- **Git** — with submodule support

## 1. Clone and Build

```bash
git clone --recurse-submodules https://github.com/jacobprall/moneypenny.git
cd moneypenny
cargo build
```

Or use the setup script, which also downloads the local embedding model:

```bash
./scripts/setup.sh
```

## 2. Initialize

```bash
mp init
```

This creates:
- `moneypenny.toml` — project configuration
- `mp-data/` — data directory with one SQLite file per agent
- A default agent named `main`

## 3. Connect to Your Agent

Moneypenny exposes its full surface area as an MCP server. One command registers
it with your agent runtime — no manual JSON editing required.

<Tabs>
  <TabItem label="Claude Code">
    ```bash
    mp setup claude-code
    ```

    Writes `.mcp.json` in your project root. For global registration across all projects:

    ```bash
    mp setup claude-code --global
    ```

    Restart Claude Code to pick up the new MCP server.
  </TabItem>
  <TabItem label="Cortex Code CLI">
    ```bash
    mp setup cortex
    ```

    Runs `cortex mcp add` to register directly. Verify with:

    ```bash
    cortex mcp list
    ```
  </TabItem>
  <TabItem label="OpenClaw">
    ```bash
    mp setup openclaw
    ```

    Writes to `~/.clawdbot/clawdbot.json`. Restart OpenClaw to pick it up.
  </TabItem>
</Tabs>

### Verify

Start a conversation and ask:

> What Moneypenny tools do you have?

Your agent should list the available MCP tools. Once connected, every operation
below can be performed by asking your agent in plain English. The following
steps show both ways.

## 4. Add Some Knowledge

Ingest a document:

<Tabs>
  <TabItem label="CLI">
    ```bash
    mp ingest path/to/document.md
    ```
  </TabItem>
  <TabItem label="Natural Language">
    Ask your MCP-connected agent:
    > Ingest the file at path/to/document.md
  </TabItem>
</Tabs>

Or from a URL:

<Tabs>
  <TabItem label="CLI">
    ```bash
    mp ingest --url "https://example.com/docs"
    ```
  </TabItem>
  <TabItem label="Natural Language">
    Ask your MCP-connected agent:
    > Ingest the docs page at https://example.com/docs
  </TabItem>
</Tabs>

## 5. Add Facts

Store structured knowledge:

<Tabs>
  <TabItem label="CLI">
    ```bash
    echo '{"op":"memory.fact.add","args":{"content":"Our API rate limit is 1000 req/min per tenant.","summary":"API rate limit: 1000 req/min/tenant","pointer":"API: rate-limit","confidence":0.95,"keywords":"api rate limit tenant"}}' \
      | mp sidecar
    ```
  </TabItem>
  <TabItem label="Natural Language">
    Ask your MCP-connected agent:
    > Remember that our API rate limit is 1000 requests per minute per tenant.
  </TabItem>
</Tabs>

List what was stored:

<Tabs>
  <TabItem label="CLI">
    ```bash
    mp facts list
    ```
  </TabItem>
  <TabItem label="Natural Language">
    Ask your MCP-connected agent:
    > List all the facts you know.
  </TabItem>
</Tabs>

## 6. Search

<Tabs>
  <TabItem label="CLI">
    ```bash
    mp facts search "rate limit"
    mp knowledge search "deployment"
    ```
  </TabItem>
  <TabItem label="Natural Language">
    Ask your MCP-connected agent:
    > What do you know about rate limits?

    > Search the knowledge base for anything about deployment.
  </TabItem>
</Tabs>

## 7. Chat (Requires LLM)

For interactive conversation with your agent:

<Tabs>
  <TabItem label="CLI">
    ```bash
    mp chat
    ```
  </TabItem>
  <TabItem label="Natural Language">
    If you're already in an MCP-connected client, you're chatting! Just talk:
    > What do you know about our deployment pipeline?
  </TabItem>
</Tabs>

<Aside type="note">
  Chat and send commands require an LLM provider. The default config uses
  Anthropic as the generation provider. To use a local model or different API provider,
  edit `moneypenny.toml` and set `provider = "local"` or `provider = "http"` with
  your API base URL and key.
</Aside>

For a one-shot message:

<Tabs>
  <TabItem label="CLI">
    ```bash
    mp send main "What do you know about our deployment pipeline?"
    ```
  </TabItem>
  <TabItem label="Natural Language">
    Ask your MCP-connected agent:
    > Ask the main agent what it knows about our deployment pipeline.
  </TabItem>
</Tabs>

## 8. Start the Gateway

For multi-agent, multi-channel operation:

<Tabs>
  <TabItem label="CLI">
    ```bash
    mp start
    ```
  </TabItem>
  <TabItem label="Natural Language">
    Ask your MCP-connected agent:
    > Start the Moneypenny gateway.
  </TabItem>
</Tabs>

This spawns worker processes for each agent, starts the scheduler, and binds
configured channel adapters (HTTP, Slack, Discord, Telegram).
