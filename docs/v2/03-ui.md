# UI Architecture

## Stack

| Layer | Choice | Rationale |
|-------|--------|-----------|
| Framework | React 19 | Ecosystem, shadcn compatibility |
| Routing | TanStack Router | Type-safe, loader pattern, search-param state |
| RPC | Hono RPC client | Zero extra deps, type-safe via Hono server inference |
| Server state | TanStack Query | Cache, invalidation, mutation lifecycle |
| Tables | TanStack Table | Sorting, filtering, column visibility, pagination |
| Components | shadcn/ui | Unstyled primitives, full control |
| Styling | Tailwind CSS | Utility-first, pairs with shadcn |
| Streaming | SSE (one-direction) | HTTP-native, simple reconnection |
| Build | Vite | HMR in dev, bundled to `apps/ui/dist` for prod |
| Theme | Dark only, retro-arcade | See `10-design-language.md` |

## Serving

Single Hono process:

```
localhost:4966/
├── /api/rpc/*              # Hono RPC (queries + mutations)
├── /api/sse/sessions/:id   # per-session event stream
├── /api/sse/events         # global event stream (cross-session activity)
├── /assets/*               # Vite build output
└── /*                      # SPA fallback → index.html
```

In dev: Vite dev server proxied through Hono on the same port.
In prod: Hono serves static `apps/ui/dist`.

Sharp 90° corners, monospace headings, single accent color, no shadows. See `10-design-language.md`.

### Top Bar

- Wordmark `MONEYPENNY` (mono, all-caps, no rounded letters)
- ⌘K command palette (search sessions, ideas, code, blueprints; quick actions)
- ⚙ settings (drawer overlay, not a tab)

### Tab Bar

- Horizontal, sharp-edged, all-caps labels
- Status glyph per tab:
  - `●` (solid) = active
  - `▶` (running) = run in flight
  - `⏸` = paused (HITL)
  - `✓` = completed
  - `!` = failed
  - dim = archived
- Right-edge `+` opens new-session dialog
- Overflow: horizontal scroll + ⌘P quick-switcher
- Drag to reorder; persisted via `tabs.position` in DB
- Close (`×` on hover) hides tab; session unaffected
- Re-opening a session creates a new tab if not already open

### Tab Persistence

Tabs are stored server-side (see `05-data.md` `tabs` table). On app load, UI fetches `/api/rpc/tabs.list` and restores. Multiple browser windows: each window queries the same set; opening/closing in one is reflected in the other on next refresh (or via global SSE `tabs.changed` event).

### Routing

Tabs map to URLs by **active tab**:

- `/` → first tab or overview
- `/s/:sessionId` → session tab active
- `/ideas` → ideas tab active
- `/search` → search tab active

The URL only encodes the active tab; the open set lives in DB. Browser back/forward navigates the active-tab history within the open set.

### Footer (session tabs only)

- cwd (clickable → directory picker dialog)
- Active blueprint name (clickable → blueprint detail drawer)
- Session running cost
- Multi-line input with submit (`⏎` or button); `⇧⏎` for newline; `/` for slash commands

For non-session tabs (overview, ideas, search), the footer is replaced by tab-specific controls (search bar in search, "new idea" button in ideas, etc.).

## Session View

Terminal-hybrid: chat structure with inline tool/diff/terminal blocks.

### Message rendering

| Element | Style |
|---------|-------|
| User message | left-aligned, accent border-left, mono content |
| Assistant message | left-aligned, no border, markdown rendered, code-fenced blocks syntax-highlighted |
| Tool call (collapsed) | sharp-edged box: `▸ tool_name(args_summary) · 234ms` |
| Tool call (expanded) | full args + result, separated; result truncates at 2KB with `[expand]` |
| File diff | unified diff with line numbers; `+` green, `-` red; collapsible if >50 lines |
| Terminal output | mono, dim foreground, scrollable max-h-96 with `[expand]` for full |
| Child session card | inline at spawn point in conversation; shows label, status, elapsed; click → open child tab |
| Pause notice | full-width bar: `⏸ paused · {reason} · type to resume` (with quick-reply buttons if `options` present) |

Tool call states: `pending` (dotted border) · `running` (animated border-pulse, no spin) · `success` (solid) · `error` (red border, expanded by default).

### Streaming events consumed

| Event | UI effect |
|-------|-----------|
| `run.started` | new run group container, scroll to bottom if pinned |
| `message.started` | new message bubble, role+timestamp set |
| `message.token` | append text to current message |
| `message.completed` | finalize markdown rendering, syntax highlight code |
| `tool.started` | tool call card in pending state |
| `tool.token` | (optional) live arg/result accumulation |
| `tool.completed` | finalize card |
| `tool.failed` | error state on card |
| `child.spawned` | child session card inserted inline |
| `child.completed` | update child card status |
| `status.changed` | update tab glyph; if paused, show pause notice |
| `config.changed` | update footer (cwd, blueprint, model) |
| `run.completed` | close run group; update session cost |
| `error` | toast + error block in message stream |

### Reconnection

SSE connections include the last received `event.id`. On reconnect, client sends `Last-Event-ID` header; server replays events from `events` table with `id > last`. Events older than retention window are dropped silently — client invalidates the session query and refetches messages.

## Global Event Channel

`/api/sse/events` streams cross-session events for the tab bar (status indicators), overview, and activity feed. Events filtered by what the UI cares about; the channel emits everything in `events` table.

The UI maintains exactly one global SSE connection plus one per-open-session SSE connection.

## Non-Session Views

### Overview
- Stat tiles: today's cost, active sessions, pending work, total knowledge
- Active sessions table (TanStack Table): label, status, blueprint, last activity, cost
- Activity feed: last 50 events from global stream

### Ideas
- TanStack Table with frontmatter columns (status, priority, tags, linked session, modified)
- Click row → idea detail drawer with rendered markdown body
- "New idea" button → markdown editor dialog
- "Launch session from idea" button on each row → opens new-session dialog with idea linked

### Search
- Single search input, results grouped by type: Sessions / Code / Ideas / Knowledge
- Each group is a TanStack Table; click result navigates appropriately

### Settings (drawer, not a tab)
- Models (strong / fast / local + base URLs + API keys status)
- Policies (table from DB; edit goes to filesystem)
- MCP servers (configurable consumers)
- Theme is dark-arcade only (no toggle)

## Out of Scope (v2)

- Multi-window coordination beyond best-effort SSE refresh
- Mobile / narrow viewport (assume ≥ 1024px)
- Light theme (dark only by design)
- i18n (English only)
