# Refactoring Techniques

Refactoring techniques describe actual refactoring steps. Most refactoring techniques have their pros and cons. Therefore, each refactoring should be properly motivated and applied with caution.

## Categories

### Composing Methods
Streamline methods, remove code duplication, and pave the way for future improvements. Excessively long methods are the root of all evil.

→ See [composing-methods/](composing-methods/)

### Moving Features between Objects
Safely move functionality between classes, create new classes, and hide implementation details from public access.

→ See [moving-features/](moving-features/)

### Organizing Data
Help with data handling, replacing primitives with rich class functionality. Untangle class associations to make classes more portable and reusable.

→ See [organizing-data/](organizing-data/)

### Simplifying Conditional Expressions
Combat conditional logic that gets more and more complicated over time.

→ See [simplifying-conditionals/](simplifying-conditionals/)

### Simplifying Method Calls
Make method calls simpler and easier to understand. Simplify the interfaces for interaction between classes.

→ See [simplifying-method-calls/](simplifying-method-calls/)

### Dealing with Generalization
Move functionality along the class inheritance hierarchy, create new classes and interfaces, replace inheritance with delegation and vice versa.

→ See [generalization/](generalization/)
